# RiskLens AI

An AI-powered project risk analyzer built for PMOs and project managers. Enter your project's key parameters and get a structured, professional risk assessment — risk score, executive summary, top risks, a probability/impact matrix, next actions, and a project health verdict — generated by an AI acting as a senior PMO consultant.

No backend, no database. This is a static React app that calls the Claude or OpenAI API directly from your browser using an API key you provide.

## Tech stack

- React 18 + TypeScript
- Vite
- Tailwind CSS
- Claude (Anthropic), OpenAI, or Google Gemini — your choice
- `jspdf` + `html2canvas` for PDF export

## Getting started

```bash
npm install
npm run dev
```

Then open the printed local URL (usually `http://localhost:5173`).

### Demo Mode

You don't need an API key to try RiskLens AI. If no key is configured, clicking **Analyze Project** runs entirely
in **Demo Mode**: it simulates a short analysis delay and returns a comprehensive, realistic mock risk report
generated from your actual form inputs (budget, timeline, team size, complexity, stakeholders, dependencies, and
methodology all influence the mock scores and recommendations). A "Demo Mode" banner appears on the results page
so it's always clear the report isn't from a live AI model, and the exported PDF/text report is labeled
accordingly.

### Connecting a real provider

Click **Demo Mode** / **Connect API Key** in the header, choose a provider, and paste in your API key. The key is
stored only in your browser's `localStorage` — it is never sent anywhere except directly to the provider you
chose, from your own browser.

| Provider | Get a key at | Default model |
|---|---|---|
| Claude (Anthropic) | https://console.anthropic.com | `claude-sonnet-4-6` |
| OpenAI | https://platform.openai.com | `gpt-4o` |
| Google Gemini | https://aistudio.google.com/apikey | `gemini-2.5-pro` |

## Important note on calling AI APIs from the browser

This MVP calls the AI provider directly from client-side JavaScript, which keeps the architecture simple (no server to deploy) but has two real-world implications worth knowing before you ship this further:

1. **Your API key is exposed to anyone who opens the browser dev tools.** That's fine for a personal tool, a local demo, or Demo Mode itself (which needs no key at all), but if you deploy this publicly with a real key configured, anyone could extract it from network requests. For a production SaaS, add a thin backend (or a serverless function) that holds the key server-side and proxies the request.
2. **CORS**: Anthropic's API supports direct browser calls when the `anthropic-dangerous-direct-browser-access` header is set (already wired up in `src/services/aiService.ts`). OpenAI's chat completions endpoint and Google's Generative Language API (Gemini) also allow direct browser calls with an API key. If any provider changes this policy, you'd need a small proxy server.

## Project structure

```
src/
  components/
    layout/       Header, theme toggle
    form/         Project intake form and its inputs
    results/      Risk score, executive summary, risk cards, matrix, actions, health
    states/       Empty, loading, and error states
    ui/           Reusable Button, Card, Badge primitives
    ApiKeyModal.tsx
  hooks/          useTheme, useLocalStorage
  lib/            AI prompt templates (the "Senior PMO Consultant" system prompt)
  services/
    aiService.ts     Calls Claude/OpenAI/Gemini and validates the response; routes to Demo Mode when no key is set
    mockAnalysis.ts  Input-aware mock risk report generator used by Demo Mode
  types/          TypeScript interfaces for project input and risk output
  utils/          Formatting, color mapping, PDF/report export helpers
```

## Customizing the AI's behavior

The system prompt lives in `src/lib/prompts.ts`. It defines the AI's persona (a 15+ year PMO consultant), the eight risk dimensions it evaluates, and the exact JSON schema it must return. Edit this file to change scoring calibration, add risk categories, or adjust tone. Demo Mode's heuristics live separately in `src/services/mockAnalysis.ts`, since they don't call any AI model.

## Adding another provider

Providers are defined in three places: the `AiProvider` union type and a `call*` function in `src/services/aiService.ts`, and an entry in the `PROVIDERS` array in `src/components/ApiKeyModal.tsx`. Follow the existing Gemini implementation as a template — it uses a plain REST `fetch` call, so no additional SDK dependency is required.

## Build for production

```bash
npm run build
npm run preview
```

The `build` output lands in `dist/` and can be deployed to any static host (Vercel, Netlify, Cloudflare Pages, S3, etc). Remember the API key caveat above if you're deploying somewhere other than your own machine.
